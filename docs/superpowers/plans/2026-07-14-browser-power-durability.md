# Browser Power-Loss Durability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve acknowledged lesson drafts, in-progress exam answers, and exam events across browser crashes, device reboots, network outages, and Codestead restarts, while reconciling each mutation exactly once and purging private recovery data at every session boundary.

**Architecture:** Keep PostgreSQL authoritative and add an IndexedDB outbox as the browser-durable boundary. A UI may say `Saved locally on this browser` only after its IndexedDB transaction completes, and may say `Saved to Codestead` only after the server acknowledges the same mutation ID. Lesson drafts reuse the server's existing idempotent receipt path. Exam autosaves gain an equivalent receipt table and replay contract; exam events retain their existing `client_event_id` uniqueness. Session-scoped opaque namespaces isolate users, compare-and-delete prevents stale acknowledgements from erasing newer edits, and logout, revocation, closed-book entry, or terminal exam state purges applicable records.

**Tech Stack:** Next.js App Router, React 19, TypeScript, native IndexedDB, `fake-indexeddb` for unit tests, Vitest/Testing Library, Playwright persistent browser contexts, Drizzle ORM, PostgreSQL.

## Global Constraints

- Do not claim that an unpersisted final keystroke survives power loss. The guarantee begins only when the UI says `Saved locally on this browser` or `Saved to Codestead`.
- Keep `sessionStorage` as an optional fast warm cache, but never treat it as the durable dirty-data source.
- Never put email, user ID, course title, answer text, or API secrets in an IndexedDB database/store/key name. Use the existing opaque, session-bound cache namespace.
- Persist a mutation before sending it. Reuse the same mutation ID for all retries. Delete only the exact acknowledged mutation with compare-and-delete semantics.
- Retry on actual request failures even if `navigator.onLine` remains `true`; it is only a hint.
- A terminal exam status is authoritative. If the deadline passes while the server is unreachable, Codestead grades the latest server autosave and purges local exam recovery data when terminal state is observed. It does not silently accept late local work.
- No service worker or background sync is introduced in this change.
- Every schema change must be generated by Drizzle, reviewed, and covered by a real PostgreSQL integration test.

## Acceptance Matrix

| Failure boundary | Required behavior | Proof |
|---|---|---|
| Tab/browser closes after `Saved locally on this browser` | Recover same draft/answer/event from IndexedDB on reopen | Persistent-profile Playwright scenario |
| NUC restarts after browser-local save but before server save | Browser retries same mutation when server returns | Playwright plus PostgreSQL row assertions |
| Server commits but response is lost | Retry returns receipt/deduplication; no duplicate evidence | Draft receipt test, exam receipt integration test, event uniqueness test |
| New edit occurs while old request is in flight | Old acknowledgement cannot delete new record | IndexedDB CAS unit test |
| Browser reports online while server is unreachable | Bounded retry continues without relying on `online` event | Fake-timer hook tests |
| Local durable write fails | UI warns; never claims local/server durability | Component tests |
| User signs out or session is revoked | Applicable recovery data is purged before credentials are reused | Lifecycle tests and persistent-profile browser test |
| Exam becomes terminal while offline | Latest server autosave is authoritative; local outbox is purged on terminal state | Exam hook/component test |
| Closed-book exam starts | Ordinary lesson recovery data is unavailable in exam context | Lifecycle test |
| Browser storage is inspected | Opaque namespace/keys; no identity or secret in identifiers/logs | Unit assertions and final diff inspection |
