# Privacy data inventory

**Inventory version:** `2026-07-14.v9`\
**Schema baseline:** `src/lib/db/schema.ts` through migration `0054`\
**Retention policy:** `src/lib/data-lifecycle/policy.ts` (`2026-07-14.v4`)\
**Learner export schema:** `14`\
**Scope:** the self-hosted Codestead application, PostgreSQL, encrypted object storage/backups, configured AI providers, runner, Gmail, Google OAuth, GitHub public-repository API, and Cloudflare Tunnel.

This is the engineering data map required before a pilot privacy review. It is not a substitute for local legal advice. A row covers every field in the named table unless a narrower field group is stated. Ordinary “application” access means the authenticated learner for their own records and the administrator/mentor through an allowlisted view or an explicitly audited sensitive-read workflow. Database and backup operators can technically access encrypted or stored records as described in the threat model.

## Recipient and retention legend

- **L** — the record’s learner through an authenticated first-party view/export.
- **A** — the single administrator/mentor, subject to role checks; sensitive reads require a purpose and audit where implemented.
- **S** — first-party application/background workers on the NUC.
- **R** — isolated runner VM; receives only one bounded job, never DB credentials or AI keys.
- **P** — the learner-consented external AI provider used for that request.
- **M** — configured Gmail delivery service; receives destination email and a non-sensitive rendered message.
- **G** — Google OAuth identity service; receives the normal OAuth protocol fields.
- **GH** — GitHub public API; receives only a public repository owner/name/ref request.
- **U** — an unauthenticated internet visitor; receives only an explicitly published allowlisted projection.
- **B** — encrypted local/offsite backups. Provider-key KEK is stored separately from database backups.
- **Account lifetime** — retained until administrator account deletion unless a shorter category applies.
- **Raw 12 months** — raw chat, code, AI request metadata, and AI attachments are deleted after 12 months.
- **Security 90 days** — token-free ended session history is deleted after 90 days unless incorporated into official evidence.
- **Audit ≥24 months** — administrator/security audit is retained for at least 24 months and is not automatically purged in v1.
- **Backup 7/4/12** — 7 daily, 4 weekly, and 12 monthly encrypted recovery points; after account deletion, backup expiry is operator-verified and never silently claimed.

| AI provider | Bounded curriculum/context/messages and code the learner chose to discuss; provider API key | Current provider-specific consent and external-AI disclosure required on every new call; email, legal identity, hidden tests, keys and other learners excluded | Governed by provider account/contract; production terms and zero-retention options require operator review |
| Gmail | Destination email, template text and action link when required | Non-sensitive notifications only; no code/chat/score/provider key/backup attachment | Provider delivery logs plus local 30-day terminal outbox record; configure account retention separately |
| Google OAuth | Standard OAuth identity, state, tokens/scopes | Login/link only for approved email/invitation; tokens encrypted/DB protected by host controls | Account lifetime or unlink/delete |
| GitHub public API | Public owner/repository/ref and bounded blobs | Static review only; no clone/install/hooks/Actions/execution | Only pinned URL/commit/findings retained; fetched buffers transient |
| Cloudflare Tunnel | HTTP connection/security metadata under operator account | Outbound-only origin exposure; no direct DB/runner access | Configure Cloudflare log retention before pilot |
| Encrypted backups | PostgreSQL dump, configuration manifest and object data; credential ciphertext but not KEK | Disaster recovery, age encryption, checksums, offsite client-side encryption | Backup 7/4/12; deletion expiry requires operator verification |

## Public, cohort, and email allowlists

The cohort projection may contain only an opted-in public alias, learner-selected badges/projects, coarse streak/mastery counts, and versioned capped leaderboard points. It must never contain email, timezone, precise activity, attempts, failures, assessment/exam scores, raw hours, hints, raw code/chat, provider use, keys, administrator notes, IP/device data, or hidden tests. Withdrawing `cohort_profile` cascades to `leaderboard` for future projections.

The public certificate verifier may contain only verifier ID, learner display name, course title/version, issue time, and valid/revoked state. The public portfolio may contain only the learner-confirmed display name/headline/about plus selected public GitHub projects, badge titles/descriptions, and valid certificate links. Neither projection may contain email, internal IDs, exact scores, hours/activity, attempts/failures, code/chat, provider data/keys, device/IP data, evidence hashes, administrator actors/reasons, hidden tests, or private repository details. Withdrawal removes the portfolio projection immediately; revocation changes only the public certificate state and never exposes the private reason.

Email templates may contain name, generic event state, expiry/result availability, and a first-party link. They must never contain a provider key, password/MFA/session token, code, chat excerpt, mistake, hidden test, detailed score, backup archive, or sensitive administrator reason. Template tests and the repository secret scanner enforce part of this boundary; live Gmail/DNS/bounce evidence remains a deployment gate.

## Prohibited collection and unresolved operator evidence

The launch schema intentionally has no advertising identifier, contact book, precise location, camera/microphone recording, DOB, payment card, public chat, raw keystroke capture, clipboard content, screen recording, or covert anti-cheat field. Exam focus/paste-size events are disclosed review signals and never automatic proof of cheating.

Before a real pilot, the operator must still record provider/Gmail/Google/Cloudflare retention settings, complete the external AI contract/license decision, run export/delete/backup expiry on synthetic accounts, verify production logs and backups with canary values, and obtain any India/US privacy review appropriate to the deployment. A server cannot remotely erase disconnected browser `sessionStorage`; account deletion revokes server access immediately, a later denied draft request purges the current namespace, and operators must instruct users of shared devices to sign out or clear site data. Admission/invitation history, raw exam events/answers, in-app notification history, and generic background-job retention need a future automated category or an explicitly approved exception.
